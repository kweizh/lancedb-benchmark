  - It explicitly excludes IDs already found in the prefix portion using a `id NOT IN (...)` pre-filter in LanceDB.
  - It appends these semantic results (with `source="semantic"`) to the prefix results.
- Single quotes in the prefix are properly escaped for the SQL query.
- The returned list contains exactly `k` dictionaries with keys `id`, `title`, `popularity`, and `source`.
