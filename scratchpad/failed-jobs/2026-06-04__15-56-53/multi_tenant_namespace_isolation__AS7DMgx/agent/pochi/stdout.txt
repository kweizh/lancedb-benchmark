The `TenantDB` class has been implemented in `/home/user/myproject/solution.py`. Here is a summary of every design decision:

---

### Table naming convention
Each tenant owns exactly one LanceDB table named:
```
tenant_<tenant_id>__documents
```
Isolation is enforced at the storage level — there are no per-row tenant filters. Any call to `connection.open_table(...)` for one tenant's table simply cannot return another tenant's rows.

### Schema
A fixed PyArrow schema is shared by all tenant tables:
| field | Arrow type |
|-------|-----------|
| `id` | `utf8` |
| `text` | `utf8` |
| `embedding` | `fixed_size_list<float32>[32]` |
| `created_at` | `utf8` |

### Method breakdown

| Method | Behaviour |
|--------|-----------|
| `__init__(connection, tenant_id)` | Binds the wrapper; derives the table name. |
| `create_documents_table(initial_rows)` | Creates `tenant_<id>__documents` via `conn.create_table`. Handles empty `initial_rows` by passing the schema directly. |
| `add_documents(rows)` | Iterates `conn.table_names()` *at call time* to gather ids from every other tenant's table using `table.search().select(["id"]).to_list()` (avoids fetching large embeddings). Raises `PermissionError` with a descriptive message before any write if a collision is found. |
| `search(query_vec, k)` | Calls `table.search(query_vec).limit(k).to_list()` on the tenant's own table only. Returns dicts containing at least `id` and `text`. |
| `delete_tenant()` | Calls `conn.drop_table(self._table_name)`. No other tables are touched. |
| `list_tenants(connection)` *(classmethod)* | Parses `connection.table_names()` with `_tenant_id_from_table`, returns a sorted list — always live, never cached. |

### Key guarantees verified
- ✅ Physical table-level isolation — no cross-tenant row visibility.
- ✅ `search()` exclusively queries the calling tenant's table.
- ✅ `add_documents()` raises `PermissionError` on cross-tenant id collision and leaves the target table unchanged.
- ✅ `delete_tenant()` removes only the calling tenant; other tenants remain queryable.
- ✅ `list_tenants()` reflects newly created and deleted tenants immediately.
