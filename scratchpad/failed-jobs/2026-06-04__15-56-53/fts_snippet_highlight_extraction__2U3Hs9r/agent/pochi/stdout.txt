### What it does

**`search_with_snippets(query, k, snippet_chars=120)`**

1. **FTS search** – connects to the existing LanceDB table (name from `LANCE_TABLE` env var) and runs `table.search(query, query_type="fts").limit(k).to_list()` to get BM25-ranked hits with `_score`.

2. **Match location** – `_find_match_offset()` finds the first case-insensitive occurrence of the query phrase in each body using `str.lower().find()`. Falls back to a whole-word regex search for any individual query token if the phrase isn't present verbatim.

3. **Snippet window** – `_build_snippet()` centers a `snippet_chars`-wide window on the match offset, clamped to `[0, len(body)]`. The matched text is wrapped in `<mark>…</mark>` (case-preserved from the original body). When there is no match at all, the leading `snippet_chars` characters are returned with no markup.

4. **Return value** – a list of dicts `{id, score, snippet, snippet_offset}` sorted by descending score, satisfying all acceptance criteria:
   - `len(plain_snippet) ≤ snippet_chars`
   - `body[snippet_offset : snippet_offset + len(plain_snippet)] == plain_snippet`
   - `<mark>` tags surround the exact body substring at the correct position within the window
