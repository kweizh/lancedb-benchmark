## What was built

**`search.py`** — a single-file CLI that:

1. **Derives the table name** from `$ZEALT_RUN_ID` (falls back to `"default"`), producing e.g. `docs_test1`.
2. **Is idempotent** — on first run it loads the 60 documents from `seed/docs.json`, creates the LanceDB table at `lancedb_data/`, and builds a Tantivy-backed FTS index with `use_tantivy=True` covering both `title` and `body`. On subsequent runs it opens the existing table/index.
3. **Executes the raw Tantivy query string** via `table.search(query, query_type="fts").limit(k).to_list()` — the string is passed through unmodified to Tantivy's query parser.
4. **Prints a JSON list** of `{"id": int, "title": str, "body": str}` objects ordered by descending relevance.
