### How it works

**Key insight**: `table.checkout(version)` is an **in-place** mutation of the table object (returns `None`). To avoid the two checkouts stomping on each other, the function opens **two independent `lancedb.connect()` sessions**, one checked out to `v_a` and one to `v_b`, then immediately reads both into pandas DataFrames. The on-disk table is never written.

**Diff logic** (pure pandas / Python sets):
| Step | Detail |
|---|---|
| `added` | `ids_b − ids_a` |
| `removed` | `ids_a − ids_b` |
| `modified` | For each id in `ids_a ∩ ids_b`, compare all non-id columns; float columns use native `float` equality (tolerant of numpy dtypes), string columns use `==`. Rows that differ get an `{"id", "old", "new"}` entry. |
