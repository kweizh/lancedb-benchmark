**Key design decisions**:

1. **Advisory file lock** (`fcntl.flock` with `LOCK_EX | LOCK_NB`): A per-table lock file at `/tmp/.{table_name}.lock` serializes concurrent writers. A fresh file descriptor is opened per attempt so threads don't share fds. When the lock cannot be acquired, it's treated as a recoverable conflict and retried.

2. **Artificial lock hold**: The lock is held for 5 ms (`LOCK_HOLD_SECS = 0.005`) before and during the `merge_insert` call, ensuring concurrent writers actually observe contention rather than racing through.

3. **Exponential backoff with jitter**: 4 retries with delays of ~50ms, ~100ms, ~200ms, ~400ms (5 total attempts). Each delay includes random jitter (0–10% of the base delay) to avoid thundering-herd patterns.

4. **Last-writer-wins by timestamp**: `merge_insert("id").when_matched_update_all(where="source.ts > target.ts").when_not_matched_insert_all()` ensures that a row with a logically older `ts` never overwrites a newer one, regardless of commit order.

5. **`write_attempts` logging**: Every attempt (success or failure) is logged as exactly one row with columns `batch_id` (string), `attempt_num` (int64, 0-indexed), `success` (bool), `error_msg` (string), and `ts` (int64 nanosecond timestamp). The table is auto-created on first use with a race-condition-safe check.

6. **No target table mutation**: The target table schema (`id int64, value int64, ts int64`) is never altered — only `merge_insert` upserts are performed.
