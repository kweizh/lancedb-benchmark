# Monitor LanceDB Index Health with `index_stats` ## Background LanceDB exposes per-index health information via `Table.index_stats(index_name)`. Operators commonly use it to detect when new rows have been written into a table but have not yet been merged into an existing vector index. Those rows still take part in queries (via a brute-force scan over the unindexed delta), but they degrade latency until the index is re-optimized. A reliable monitoring workflow first builds an index over a seed dataset, snapshots `num_indexed_rows` / `num_unindexed_rows`, ingests new rows, and re-reads the stats to confirm the delta grew. ## Requirements - Write a single Python program at `/home/user/project/run.py` that, when executed inside the task container, performs the full monitoring workflow described below against a real LanceDB instance. - The program MUST: 1. Connect to the LanceDB URI provided in the `LANCEDB_URI` environment variable (default `/workspace/db` when unset). 2. Create a fresh table named `points` with an `id: int64` column and a `vector: fixed_size_list<float32>[16]` column. 3. Seed the table with exactly 400 deterministic rows generated from `numpy.random.default_rng(50)` (no model inference, no GPU). 4. Build an IVF_PQ index over the `vector` column with `metric="cosine"`, `num_partitions=4`, `num_sub_vectors=4`, `replace=True`, and wait for the index to become ready (using `wait_for_index(["vector_idx"], timeout=datetime.timedelta(seconds=120))`). 5. Capture an initial snapshot of `table.index_stats("vector_idx")` (`num_indexed_rows`, `num_unindexed_rows`, `index_type`). 6. Append 50 additional deterministic rows to the table (these new rows must remain unindexed - DO NOT call `optimize()` or rebuild the index afterwards). 7. Capture a second snapshot of `table.index_stats("vector_idx")` and write the monitoring report to `/workspace/output/index_stats.json`. - The output JSON file MUST contain exactly these top-level keys: - `index_type` (string): the LanceDB-reported index type for `vector_idx`. - `initial_indexed` (int): `num_indexed_rows` from the snapshot taken immediately after the index is built. - `initial_unindexed` (int): `num_unindexed_rows` from that same initial snapshot. - `unindexed_after_add` (int): `num_unindexed_rows` from the second snapshot, taken after the 50-row append. ## Implementation Hints - Use `lancedb.connect(...)` and `db.create_table(name, schema=...)` (or `data=...`) to create the `points` table.
