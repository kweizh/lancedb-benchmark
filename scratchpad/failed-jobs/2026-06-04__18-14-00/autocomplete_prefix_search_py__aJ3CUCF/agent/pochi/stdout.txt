## What it does

### `autocomplete(prefix, k=10)`

**Step 1 – SQL prefix match**
- Reads `LANCE_DB_URI`, `MOVIES_TABLE`, and `PREFIX_TABLE` from the environment on every call.
- Runs a case-insensitive LIKE filter on the pre-seeded `title_lower` column:
  ```sql
  title_lower LIKE '<lower_prefix>%'
  ```
- Fetches all matching rows, sorts them by `popularity` descending, clips to `k`, and marks each with `source = "prefix"`.
- If `M ≥ k`, returns immediately.

**Step 2 – Semantic vector fallback (when M < k)**
- Looks up the precomputed 32-dimensional embedding from `prefix_vectors` where `prefix = lower(prefix)`.
- Builds an `id NOT IN (...)` prefilter that excludes all ids already returned in the prefix portion.
- Calls `movies_tbl.search(query_vector).where(excl_filter, prefilter=True).limit(k - M)` for default L2 nearest-neighbour search.
- Marks each result with `source = "semantic"` and strips the internal `_distance` key LanceDB may add.
- Returns the M prefix rows followed by the (k − M) semantic rows.
