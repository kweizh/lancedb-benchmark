## How it works

### `colbert_search(query_token_vecs, k=5)`

1. **M independent searches** — iterates over each of the M query token vectors (rows of the `(M, 32)` input array) and issues one `tbl.search(...).distance_type("cosine").limit(50).to_pandas()` call per token. No pooling is ever performed.

2. **Distance → similarity** — converts LanceDB's returned `_distance` to cosine similarity via `sim = 1 - distance`.

3. **MaxSim aggregation** — for each query token `q_i`, groups the 50 retrieved rows by `doc_id` and takes the **maximum** similarity (`groupby("doc_id").max()`). This is the per-query-token MaxSim. Docs absent from a particular top-50 implicitly contribute `0`.

4. **Late-interaction score** — accumulates each query token's MaxSim values into a running `doc_scores` dict (sum across all M tokens).

5. **Ranking** — sorts by descending score, ascending `doc_id` for ties, and returns the top-k `doc_id` values as `list[int]`.
