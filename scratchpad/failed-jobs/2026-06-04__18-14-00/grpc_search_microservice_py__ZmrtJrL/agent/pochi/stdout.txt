### How it works

**`proto/search.proto`** — defines the exact contract required:
```proto
syntax = "proto3";
package search;

service SearchService {
  rpc Search(SearchRequest) returns (SearchResponse);
}
message SearchRequest { repeated float vector = 1; int32 k = 2; string where_clause = 3; }
message SearchResponse { repeated Hit hits = 1; }
message Hit { int64 id = 1; float score = 2; string title = 3; }
```

**Stubs** were generated with:
```
python3 -m grpc_tools.protoc -I proto --python_out=. --grpc_python_out=. proto/search.proto
```
Both `search_pb2.py` and `search_pb2_grpc.py` land in the project root and are directly importable.

**`server.py`** — at startup opens `lancedb.connect(...).open_table("documents")` once and reuses the handle. The `Search` RPC:
- Builds a LanceDB query with `.search(vector).limit(k)`
- Appends `.where(where_clause)` only when the field is non-empty
- Maps each result row to a `Hit(id, title, score=_distance)`
- Binds to `0.0.0.0:50051` via `grpc.server(ThreadPoolExecutor(max_workers=4))`
