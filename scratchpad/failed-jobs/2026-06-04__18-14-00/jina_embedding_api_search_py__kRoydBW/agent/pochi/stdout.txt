### `solution.py`
- **`build_index() -> None`**  
  - Loads all 50 headlines from `headlines.json`  
  - Embeds them in a single Jina API call with `task="retrieval.passage"`  
  - Creates (or overwrites) a LanceDB table named `headlines_${ZEALT_RUN_ID}` under `lancedb_data/`  
  - Schema: `id` (int64), `headline` (str), `topic` (str), `vector` (fixed-size float32 list matching the API's returned dimension)

- **`search(query, k=5, task="retrieval.query") -> list[dict]`**  
  - Embeds the query via Jina using the given `task`  
  - Runs `tbl.search(query_embedding).limit(k).to_list()` against the LanceDB table  
  - Returns exactly `k` dicts each containing `id` (int), `headline` (str), `topic` (str)

### Key design decisions
| Concern | Approach |
|---|---|
| Jina auth | `Authorization: Bearer $JINA_API_KEY` header |
| HTTP client | `httpx.post(...)` with 120 s timeout |
| Batching | All 50 passages in **one** API call for minimal latency |
| Embedding dim | Detected dynamically from the first API response |
| Table naming | `f"headlines_{os.environ['ZEALT_RUN_ID']}"` — collision-safe across concurrent runs |
| Overwrite | Existing table is dropped before re-creation on `build_index()` |
