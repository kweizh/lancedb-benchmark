## What was built

### `build_dataset()`
1. Seeds `numpy.random.default_rng(2026)`, draws a 32×32 standard-normal matrix `A`, and computes `Q, _ = np.linalg.qr(A)` — giving 10 mutually-orthogonal unit-vector centroids in `Q[:, c]`.
2. For each cluster `c ∈ [0,9]` and slot `j ∈ [0,11]`, draws a fresh 32-d noise vector and forms `row_vec = Q[:,c] + 0.05*noise` (cast to `float32`).
3. Writes a 120-row LanceDB table (`mmr_docs_${ZEALT_RUN_ID}`) at `/app/db` with schema `{id: int64, cluster_id: int64, vector: fixed_size_list<float32>[32]}` using `mode="overwrite"`.

### `mmr_search(query_vec, k=10, lambda_=0.5)`
1. Fetches the **top-30 candidate pool** with `.search(q).distance_type("cosine").limit(30).to_list()`.
2. Pre-normalises all 30 candidate vectors and the query vector (float64) to compute:
   - `rel_scores[i]` = cosine similarity of candidate `i` to the query (shape 30)
   - `sim_matrix[i,j]` = pairwise cosine similarity among candidates (30×30)
3. Greedy MMR loop: each iteration picks `argmax(λ·sim(q,d) − (1−λ)·max_{d'∈S} sim(d,d'))`, using `np.ix_` slicing on the pre-computed matrix — no extra LanceDB calls.
4. Returns selected `id` values in pick order as `list[int]` of length `k`.
