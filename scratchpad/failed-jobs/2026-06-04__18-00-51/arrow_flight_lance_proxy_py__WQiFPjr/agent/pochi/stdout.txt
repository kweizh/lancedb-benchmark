## Implementation

```
FlightProxyServer(FlightServerBase)
 тФФтФА __init__   тАУ connects to LanceDB and opens the existing `documents` table once at startup
 тФФтФА do_get     тАУ decodes the UTF-8 JSON ticket, runs table.search(vector).limit(k).to_arrow(),
                  selects the four required columns, and returns a RecordBatchStream
main()         тАУ binds to grpc://0.0.0.0:8815 and calls server.serve()
```

Key design choices:
- **No re-seeding / recreation** тАФ `lancedb.connect` + `open_table` only opens the pre-existing table.
- **Column selection** тАФ `result.select(["id", "text", "embedding", "_distance"])` guarantees the exact four columns in a fixed order regardless of what LanceDB returns internally.
- **No auth / TLS / middleware** тАФ plain gRPC as required.
