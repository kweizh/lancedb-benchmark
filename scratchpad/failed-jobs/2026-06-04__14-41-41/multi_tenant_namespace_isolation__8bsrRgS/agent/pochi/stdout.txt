### Architecture Summary

The implementation leverages LanceDB's physical table separation to enforce strict tenant isolation. The table name for each tenant is uniquely derived as `tenant_{tenant_id}`.

* **Constructor**: Binds the wrapper to a specific tenant with `TenantDB(connection, tenant_id)`.
* **Table Creation**: `create_documents_table(initial_rows)` provisions the tenant's physical table using a strict PyArrow schema (including a 32-dimensional float vector for `embedding` and an ISO-8601 string for `created_at`).
* **Cross-Tenant Collision Prevention**: Both `create_documents_table` and `add_documents` inspect all other existing tenant tables in the connection before inserting/modifying any records. If any incoming `id` already exists in another tenant's namespace, a `PermissionError` is raised, and the target table remains completely unchanged.
* **Vector Similarity Search**: `search(query_vec, k)` queries the tenant's own table using LanceDB's native vector similarity search, returning at most `k` documents ordered by similarity.
* **Tenant Listing**: `TenantDB.list_tenants(connection)` extracts and returns a sorted list of all active tenant IDs immediately from `connection.table_names()`.
* **Tenant Deletion**: `delete_tenant()` removes the tenant's table from the connection without affecting any other tenants' tables.
