LanceDB only allows **one vector index per column per table**, so to study the recall-quality trade-off you must either build the same data into multiple tables or drop and recreate the index between sweeps. You are given a deterministic 1024-row, 64-dim numpy dataset and 30 fixed query vectors. For each query, recall\@10 is `|candidate_top10 ∩ brute_force_top10| / 10`. Report the mean over the 30 queries. * Provide a CLI entrypoint at `/home/user/myproject/run_sweep.py` that calls `sweep()` and writes the result to `/home/user/myproject/result.json` as a JSON object with string keys (e.g. `{"4": 0.78, "8": 0.91, "16": 0.97}`). * Append `${ZEALT_RUN_ID}` (read from the `ZEALT_RUN_ID` environment variable) to every LanceDB table name you create, so concurrent runs don't collide. ## Implementation Hints * LanceDB enforces a single vector index per column per table.
