### Key Components:
- **`indexer.py`**: The main script that handles both indexing and searching.
  - **`index_docs()`**: Walks through `/app/docs/` recursively, parses Markdown files into sections, computes embeddings using OpenAI's `text-embedding-3-small` model, and persists them into a LanceDB table.
  - **`search(query, k)`**: A Python API that embeds a natural-language query and retrieves the top `k` most relevant sections from the LanceDB table.
  - **Markdown Parsing**: Uses regex to extract the top-level `# Title` and split the file into sections based on `## Section` headers.
  - **Idempotency**: The indexing process uses `mode="overwrite"` in LanceDB to ensure that re-running the script with the same `ZEALT_RUN_ID` updates the index without errors.
  - **Dynamic Table Naming**: The table name automatically incorporates the `ZEALT_RUN_ID` environment variable as required.

