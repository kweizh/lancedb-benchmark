## Implementation

**`expanded_search(query, k=10) → list[int]`** works in four steps:

1. **Connect lazily** — opens the LanceDB database at `$LANCEDB_URI` and the table at `$LANCEDB_TABLE` once, caching both at module level.

2. **Build the FTS index once** — calls `table.create_fts_index("content", use_tantivy=False, replace=False)` on the first invocation; the `replace=False` + bare `except` pattern makes subsequent calls a no-op without error.

3. **Expand the query** — lowercases the query, then for each whitespace-separated token iterates through `wn.synsets(token)` and collects up to 3 single-token lemmas (no `_` or spaces), deduplicating globally. The original token always heads its expansion group.

4. **Execute a SHOULD boolean FTS query** — wraps each expanded term in a `MatchQuery("content")` and combines them with `BooleanQuery([(Occur.SHOULD, ...)])`, then calls `.search(bool_query, query_type="fts").limit(k).to_list()` and returns the `id` fields as `list[int]`.
