Implemented `/home/user/project/recommend.py` — a CLI script that personalizes search results by blending user taste with a query vector.

**How it works:**

1. **Loads user history**: Queries the `user_history` LanceDB table for up to 10 most recent interactions for the given `--user-id`, sorted by `ts` descending.

2. **Computes taste vector**: Joins interacted `item_id`s to the `items` table to retrieve their 64-d embeddings, then computes the mean as the "user taste vector".

3. **Blends vectors**: Uses formula `(1 - alpha) * query + alpha * taste` where `alpha` comes from the `BLEND_ALPHA` environment variable (default `0.3`). Falls back to the pure query vector when the user has no history.

4. **Vector search**: Runs LanceDB's `search()` on the `items` table against the blended vector, excluding already-seen items via a `NOT IN` SQL filter on `id`.

5. **Output**: Writes the top-k item IDs as a JSON array of integers to the `--output` path.

**Usage:**
```
python3 recommend.py --user-id u_test --query-vec query.npy --k 5 --output results.json
BLEND_ALPHA=0.7 python3 recommend.py --user-id u_test --query-vec query.npy --k 10 --output results.json
```

**Verified with tests:**
- Users with history return correct personalized results with no already-seen items
- Users with no history fall back to pure query vector search
- Custom `BLEND_ALPHA` values work correctly
- Output is always a JSON array of integer IDs with length matching `--k`
