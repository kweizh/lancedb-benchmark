### What was built

`expanded_search(query, k)` works as follows:

1. **Lazy initialisation** — on the first call it connects to the LanceDB database at `$LANCEDB_URI`, opens the table named by `$LANCEDB_TABLE`, and creates a native FTS index (`use_tantivy=False, replace=False`). Subsequent calls reuse the same handles and skip index creation silently.

2. **WordNet expansion** — every whitespace-separated token in the lowercased query is passed through `wn.synsets(token)`. Single-token lemmas (no `_` or spaces) are collected until at most 3 synonyms per original token are gathered. The original term is always first.

   ```
   car  → ['car', 'auto', 'automobile', 'machine']
   ```

3. **Boolean FTS query** — all unique expanded terms are wrapped in a `BooleanQuery` of `(Occur.SHOULD, MatchQuery(term, "content"))` clauses and fired against the table via `table.search(fts_query, query_type="fts").limit(k)`.

4. **Return value** — `list[int]` of document IDs in FTS-score descending order, length ≤ k.
