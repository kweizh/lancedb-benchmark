### Summary of Implementation
1. **WordNet Query Expansion**:
   - The query is first lowercased and split into whitespace-separated tokens.
   - For each token, we look up its synonyms from the pre-downloaded offline WordNet corpus.
   - We filter out synonyms that are not single-token (containing `_` or whitespace) or are duplicates of already collected terms.
   - We keep at most **3** synonyms per token, while ensuring the original token itself remains in the expansion.

2. **LanceDB Native FTS**:
   - The database is opened from the path specified in the `LANCEDB_URI` environment variable, using the table name from `LANCEDB_TABLE`.
   - A native LanceDB FTS index (`use_tantivy=False`) is built on the `content` column if it does not yet exist. We check for its existence dynamically using `table.list_indices()` to prevent errors on subsequent calls.

3. **Boolean Query Construction**:
   - We construct a structured `BooleanQuery` using `MatchQuery` and `Occur.SHOULD` for each of the expanded terms, representing an `OR` query.
   - We execute the search, limit the results to `k`, and return the document IDs as a list of Python `int` sorted by FTS score descending.
