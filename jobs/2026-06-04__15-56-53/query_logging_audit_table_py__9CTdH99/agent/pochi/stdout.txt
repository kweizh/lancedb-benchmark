### What was built

**`LoggedSearcher`** — a thin, observable wrapper around a LanceDB articles table.

#### Constructor
```python
LoggedSearcher(db_uri: str, articles_table: str, logs_table: str)
```
Opens the LanceDB database and the target articles table eagerly; the audit table is opened/created **lazily** on the first `.search()` call.

#### `search(query_vector, top_k, query_id, user_id, query_text="") → list[dict]`
1. Times `table.search(query_vector).limit(top_k).to_list()` with `time.perf_counter()` and converts the delta to milliseconds.
2. Writes exactly one audit row to the `query_logs` table (creating it from the explicit PyArrow schema if it doesn't exist yet).
3. Returns the raw hit list unchanged, so callers get identical results to a direct LanceDB search.

#### Audit-table schema (`query_logs`)

| Column | Arrow type | Notes |
|---|---|---|
| `query_id` | `string` | passed verbatim |
| `user_id` | `string` | passed verbatim |
| `query_text` | `string` | passed verbatim |
| `ts` | `timestamp("us", tz="UTC")` | wall-clock UTC at logging time |
| `latency_ms` | `float64` | strictly > 0 |
| `hit_count` | `int64` | == `len(top_ids)` |
| `top_ids` | `list<int64>` | `id` values of hits in rank order |
